# Weather-AI 챗봇 질문/답변 시나리오

## 1. 문서 목적

본 문서는 Weather-AI 프로젝트의 AI 챗봇 기능에서 사용자가 입력할 수 있는 질문 유형과 그에 따른 챗봇 응답 시나리오를 정리한 문서이다.

Weather-AI 챗봇은 사용자의 자연어 질문에서 핵심 키워드를 추출하고, 해당 키워드에 맞는 intent를 분류한 뒤, intent별 응답 함수를 통해 답변을 제공한다.

이 문서는 다음 목적을 가진다.

- 챗봇이 처리할 수 있는 질문 범위 정리
- 질문 유형별 키워드와 intent 매핑
- 테스트 질문 목록 정리
- 발표 및 시연 시 챗봇 기능 설명 자료로 활용
- 향후 챗봇 기능 확장 시 기준 문서로 활용

---

## 2. 챗봇 기능 개요

Weather-AI 챗봇은 악천후 대응 AI 기반 위험 차량 탐지 및 관제 시스템에서 사용자와 관리자의 질문에 응답하는 기능이다.

챗봇은 단순한 자유 대화보다는 Weather-AI 서비스와 관련된 안내형 챗봇에 가깝다.  
사용자는 챗봇을 통해 서비스 소개, 탐지 차량, 날씨 조건, 위험도 점수, 오탐 방지 방식, CCTV 확인 방법, 알림 기준, 특정 지역 위험 상태 등을 질문할 수 있다.

챗봇의 기본 동작 흐름은 다음과 같다.

```text
사용자 질문 입력
→ 질문 문장 전처리
→ 키워드 기반 intent 분류
→ intent에 맞는 응답 함수 실행
→ 챗봇 답변 반환

## 3. 챗봇 intent 목록 

| intent            | 의미                          | 설명                          |
| ----------------- | ----------------------------- | --------------------------- |
| project_help      | 프로젝트 소개                 | Weather-AI가 어떤 서비스인지 설명     |
| vehicle_help      | 탐지 차량 안내                | 탐지 대상 차량과 위험 차량 기준 설명       |

| risk_score_help   | 위험도 점수 / 오탐 방지 안내   | 위험도 점수 산출 방식과 LLM 검증 설명     |
| weather_api_help  | 기상청 API / 악천후 판단 안내  | 날씨 정보 수집과 감시 활성화 기준 설명      |
| yolo_help         | YOLO / 차량 탐지 방식 안내    | 차량 탐지 모델과 바운딩박스, 신뢰도 설명     |
| admin_help        | 관리자 기능 안내              | 관리자 대시보드, 통계, 알림, 사용자 조회 설명 |
| cctv_help         | CCTV / 지도 안내              | CCTV 확인 위치와 지도 기반 이벤트 확인 설명 |
| notification_help | 알림 기준 안내                | 위험 알림 발생 기준과 관리자 알림 설명      |
| board_help        | 위험 이벤트 게시판 안내       | LLM 제목/본문 생성과 게시판 자동 등록 설명  |
| service_help      | 서비스 사용법 안내             | Weather-AI 서비스 이용 방법 설명     |
| risk_status       | 특정 지역 위험 상태 조회       | 지역명 또는 도로명을 기준으로 위험 알림 조회   |
| unknown           | 알 수 없는 질문                | 등록되지 않은 질문에 대한 안내           |

위 순서대로 질문하면 Weather-AI의 전체 흐름을 자연스럽게 설명할 수 있다. 

## 7. 챗봇 응답 설계 기준
Weather-AI 챗봇 응답은 다음 기준을 따른다. 

## 7-1. 프로젝트 범위 안에서만 답변
챗봇은 Weather-AI 프로젝트와 관련된 질문에 집중한다.
프로젝트와 관련 없는 질문은 unknown 응답으로 처리한다.
예시:
사용자: 밥 뭐 먹지?
챗봇: 죄송해요. 아직 해당 질문은 정확히 이해하지 못했습니다. Weather-AI 소개, 탐지 차량, 위험도 점수, 오탐 방지, CCTV 확인, 알림 기준, 특정 지역 위험 상태에 대해 질문해 주세요. 

## 7-2. 사용자 질문이 짧아도 의도 추정
사용자가 "오탐", "YOLO", "기상청"처럼 짧게 입력해도 관련 intent로 분류할 수 있도록 키워드를 구성한다.

사용자: 오탐
intent: risk_score_help

사용자: YOLO
intent: yolo_help

사용자: 기상청
intent: weather_api_help

## 7-3. 위험 상태 질문은 지역명 포함 필요 
특정 지역이나 도로의 위험 상태를 조회하려면 지역명 또는 도로명이 필요하다.

예시:

사용자: 위험해?
챗봇: 확인할 지역명이나 도로명을 함께 입력해 주세요. 예: '강남대로 위험해?'

## 7-4 실제 위험 알림 데이터가 없을 수 있음

지역 위험 상태 조회는 DB에 저장된 탐지 이벤트를 기반으로 한다.
따라서 해당 지역에 위험 알림이 없으면 “현재 관련 위험 알림은 확인되지 않았습니다.”라고 답변한다.

예시:

사용자: 강남대로 위험해?
챗봇: 현재 강남대로 관련 위험 알림은 확인되지 않았습니다.

## 8. 챗봇 기능 확장 방향

현재 챗봇은 키워드 기반 intent 분류 방식으로 동작한다.
향후에는 다음과 같은 방향으로 확장할 수 있다.

| 확장 기능             | 설명                                    |
| ----------------- | ------------------------------------- |
| 자연어 intent 분류 고도화 | 단순 키워드가 아니라 LLM을 활용해 질문 의도를 더 유연하게 분류 |
| 위치 기반 개인화 답변      | 사용자의 현재 위치 또는 선택한 지역을 기준으로 위험 상태 안내   |
| 실시간 CCTV 이벤트 요약   | 최근 탐지 이벤트를 요약하여 사용자에게 제공              |
| 관리자 전용 챗봇 기능      | 관리자 권한이 있는 사용자에게 통계, 알림, 이벤트 처리 현황 안내 |
| 알림 상세 연결          | 챗봇 답변에서 위험 이벤트 상세 페이지 또는 지도 위치로 이동    |
| 다중 질문 처리          | “강남 위험해? 알림은 언제 와?”처럼 여러 의도가 섞인 질문 처리 |

## 9. 설명 문장

Weather-AI 챗봇은 사용자의 질문에서 핵심 키워드를 추출하여 intent를 분류한다.

예를 들어 사용자가 “오탐 방지는 어떻게 해?”라고 질문하면 챗봇은 오탐, LLM 검증, 2차 검증 등의 키워드를 기준으로 risk_score_help intent로 분류하고, 1차 위험도 점수 필터링과 2차 LLM 검증을 통해 오탐 가능성을 줄인다는 답변을 제공한다.

또한 “YOLO는 뭐 하는 거야?”라는 질문은 yolo_help로 분류하여 차량 탐지 모델의 역할을 설명하고, “기상청 API는 왜 사용해?”라는 질문은 weather_api_help로 분류하여 날씨 정보 수집과 악천후 판단 과정을 안내한다.

이처럼 챗봇은 서비스 소개, 탐지 차량, 날씨 조건, 위험도 점수, 오탐 방지, 기상청 API, YOLO 탐지 방식, 관리자 기능, CCTV 확인, 알림 기준, 위험 이벤트 게시판, 특정 지역 위험 상태 조회에 대한 답변을 제공한다.

## 10. 정리 
Weather-AI 챗봇은 단순한 대화형 챗봇이 아니라, 프로젝트 기능을 사용자가 쉽게 이해할 수 있도록 돕는 안내형 챗봇이다.

챗봇은 사용자의 질문을 키워드 기반으로 분석하여 intent를 분류하고, intent에 맞는 답변을 제공한다.
이를 통해 사용자는 Weather-AI의 주요 기능과 위험 차량 탐지 기준, 위험도 산출 방식, 알림 기준, CCTV 확인 방법 등을 자연어로 안내받을 수 있다.
